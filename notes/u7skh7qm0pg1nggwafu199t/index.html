For a minimum length code, length of symbols should be in the opposite order of the probability of the symbol occuring.

We want a instantaneous, uniquely decodable code for given set of input symbols sis_isi​, along with their probabilities pip_ipi​.

Theorem

If pip_ipi​ are in descending order, then lil_ili​ should be in ascending order

p1≥p2≥...≥pqp_1 \geq p_2 \geq ... \geq p_qp1​≥p2​≥...≥pq​
l1≤l2≤...≤lql_1 \leq l_2 \leq ... \leq l_ql1​≤l2​≤...≤lq​

Proof

Suppose pip_ipi​ are in order, but at least one pair of lil_ili​ are not.

Their contribution to the average length LLL is

before=pnln+pmlmbefore = p_nl_n + p_ml_mbefore=pn​ln​+pm​lm​

After interchanging the two codes

after=pnlm+pmlmafter = p_nl_m + p_ml_mafter=pn​lm​+pm​lm​

Difference is

before−after=pn(ln−lm)+pm(lm−ln)=(pn−pm)(ln−lm)before - after = p_n(l_n - l_m) + p_m(l_m - l_n) = (p_n - p_m)(l_n - l_m)before−after=pn​(ln​−lm​)+pm​(lm​−ln​)=(pn​−pm​)(ln​−lm​)

One of these two is factors is negative, hence the difference is negative. So if we interchange, we get a smaller contribution to the average length LLL. This means that in a minimum length code, the two running inequalities must be true.
